数学表达式计算方法

对于一个以字符串形式输入含有四则运算和括号的数学表达式，如“10+(2*(236/2-1)+2)*9”，其形式称为中缀表达式，这是对人类而言直观的表现形式，但是对机器而言，其结构和规则因为括号和运算符优先级的存在而变得复杂，因此编程计算时，需要将中缀表达式转化为后缀表达式（逆波兰式）。

后缀表达式

后缀表达式，又称逆波兰式，指的是不包含括号，运算符放在两个运算对象的后面，所有的计算按运算符出现的顺序，严格从左向右进行（不再考虑运算符的优先规则）。

对于这样一个式子其运算就相当简单了，按如下算法即可：

  1. 初始化一个空的栈
  2. 从左至右扫描表达式
    • 遇到数字：压栈
    • 遇到运算符：将栈最顶上的两个数字弹出，进行该运算符代表的操作（此处需注意先弹出的数字是第二个运算数，后弹出的数字是第一个运算数）；将所得结果压栈
  3. 最终栈中只有一个元素，即表达式的运算结果

中缀表达式转后缀表达式

因为后缀表达式的计算十分简单，因此这个问题的难点就在于如何将中缀表达式转化为后缀表达式，在此，还是可以利用栈这一数据结构完成这个操作：

  1. 初始化空栈，初始化空的列表
  2. 从左到右扫描中缀表达式：
    • 遇到数字：加入列表队尾
    • 遇到左括号：压栈
    • 遇到运算符：比较该运算符比栈顶符号的优先级
      • 该运算符优先级高：压栈
      • 其他：依次弹出栈顶符号加入列表队尾，直到栈顶符号优先级低于该运算符（或为空），此时将该运算符压栈
    • 遇到右括号：依次弹出栈顶元素并将它加入列表队尾，直到栈顶元素为左括号，此时将左括号出栈（并不加入队尾）
  3. 列表中就是等价的后缀表达式了

关于栈中可能出现的五种符号的优先级，定义如下：

+ - * / (
+ = = < < >
- = = < < >
* > > = = >
/ > > = = >
) < < < < =

自定义运算符

对于自定义运算符，题目反而变得简单了，因为不用考虑运算符之间的优先级，上述表格也完全简化了。

只需首先实现一个函数，接受两个运算数作为参数（有顺序），根据运算符的定义，完成相应的计算并返回舍入后的结果；其次对表达式按上述方法先转为后缀式再计算，每一步计算调用刚才实现的函数即可。
